Abstract

Large language models (LLMs) face inherent limitations in knowledge access and factuality, constrained by their parametric knowledge representations. While retrieval-augmented generation (RAG) has emerged as a solution, it suffers from context window pollution, reduced reasoning capacity, and unnatural integration of external information.

We propose Bridge Neural Networks (BNNs), a novel architecture that repurposes a subset of neurons to create dedicated neural pathways for external knowledge access. Unlike RAG, BNNs detect knowledge boundaries through trained neuron activations, generate neural query representations, and integrate external information directly at the hidden state level without consuming context tokens.

We present a theoretical foundation for BNNs, detail their architecture, outline training methodology, and propose evaluation frameworks that measure factuality, reasoning preservation, and integration quality. Our analysis suggests BNNs offer a more elegant and efficient approach to knowledge integration that preserves model reasoning capacity while enabling selective external information access.

Key Innovations

Bridge Architecture: Dedicated neural pathways for external knowledge access without context window pollution.

Information Flow: Comparison of traditional RAG vs Bridge Neural Networks showing selective activation and neural integration.

Smart Allocation: Pruning-guided bridge neuron allocation repurposes low-importance neurons for knowledge access.

Mathematical Foundation: Formal framework for bridge activation, query generation, and information integration.

Training Methodology: Multi-phase curriculum for teaching knowledge boundary detection and integration.

Implementation

Bridge Neural Networks can be implemented as adapters for existing pretrained models or integrated into new architectures. Our reference implementation includes:

  • Bridge Adapter: Pluggable module for existing transformers
  • Training Pipeline: Multi-phase curriculum with specialized loss functions
  • Knowledge Service: External knowledge integration with vector databases and APIs
  • Evaluation Framework: Comprehensive metrics for knowledge integration quality
